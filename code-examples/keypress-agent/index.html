KeypressAgent Example

The Keypress Agent is a simple module to control and manage key triggers and hot key shortcuts for web application under the concept of layering. This example demonstrates the idea behind layering and the simplicity of using the keypress agent.

The Source Code

<script type="text/javascript"> var mainScreenLayer = KeypressAgent.createLayer(); mainScreenLayer .add(13, function() { alert('Main screen enter key pressed'); }) .add('a', function() { alert('Main screen key combination pressed'); }, {ctrl: true}); var secondaryScreenLayer; function openSecondaryScreen() { secondaryScreenLayer = KeypressAgent.createLayer(); secondaryScreenLayer .add(115, function() { alert('Secondary screen F4 key pressed'); }) .add('z', function() { alert('Secondary screen key combination pressed'); }, {ctrl: true, alt: true}); $("#secondary-screen").show(); } function closeSecondaryScreen() { secondaryScreenLayer.destroy(); $("#secondary-screen").hide(); } </script> <div class="radius" id="main-screen" style="padding: 20px; background: #ddd;"> <h4>Main Screen</h4> <p>This is the main screen of our example web application. It reacts to the following keypress events:</p> <ul> <li><code class="inline">enter</code> - alerts 'Main screen enter key pressed' message</li> <li><code class="inline">ctrl+a</code> - alerts 'Main screen key combination pressed' message</li> </ul> <p>Once tried to interact with the screen keypress options, open the next screen by clicking the button below:</p> <button class="btn btn-primary" onclick="openSecondaryScreen()">Open Next Screen</button> </div> <div id="secondary-screen" style="display: none; background: black; background: rgba(0,0,0,.92); position: fixed; top: 0; bottom: 0; right: 0; left: 0; width: 100%; height: 100%; padding: 30px; color: white; z-index: 99999999;"> <h4>Secondary Screen</h4> <p>This is a secondary screen of our example web application. Note that once it is open, the entire keypress functionality of the main screen is disabled.</p> <ul> <li><code class="inline">F4</code> - alerts 'Secondary screen F4 key pressed' message</li> <li><code class="inline">ctrl+alt+z</code> - alerts 'Secondary screen key combination pressed' message</li> </ul> <p>Once tried to interact with this screen, close it and return to the main screen by clicking the button below. Note that once the secondary screen is closed, the main screen keypress functionality is enabled again</p> <button class="btn btn-primary" onclick="closeSecondaryScreen()">Close Secondary Screen</button> </div>

The Result

Main Screen

This is the main screen of our example web application. It reacts to the following keypress events:

  • enter - alerts 'Main screen enter key pressed' message
  • ctrl+a - alerts 'Main screen key combination pressed' message

Once tried to interact with the screen keypress options, open the next screen by clicking the button below:

Secondary Screen

This is a secondary screen of our example web application. Note that once it is open, the entire keypress functionality of the main screen is disabled.

  • F4 - alerts 'Secondary screen F4 key pressed' message
  • ctrl+alt+z - alerts 'Secondary screen key combination pressed' message

Once tried to interact with this screen, close it and return to the main screen by clicking the button below. Note that once the secondary screen is closed, the main screen keypress functionality is enabled again
