DOM Agent With a jQuery Plugin

To get a good understanding of how the DOM Agent could help us and make our development process simpler and more modular, let's look at a very common example of certain element that demand some Javascript operations when appear on our DOM.

A good example of that will be an element that we want to apply some jQuery plugin in. For example, let's take jQuery UI Accordion plugin (or similarly any other jQuery plugin that demands some initialization). First of all, let's note that the way to initialize the simplest Accordion element, we should call the jQuery.accordion method like so: $("some-element").accordion();. Next, let's we have a page with one accordion element, and a button below. Each time the button is clicked, a new accordion element is inserted to the DOM. Each time that happens, we have to initialize the element once again.

The solution offered by the DOM Agent, is to define a constructor function to elements, based on a selector matching. The example below, shows the page containing dynamic accordion elements, each gets initialized by its constructor the moment it gets inserted to the DOM.

Conclusions & Notes

  • First of all note that the use of jQuery UI or the use of the accordion plugin are merely an example, this would give you a good modularity and control over your code in any case you want to run some initialization to your DOM elements and you dynamically insert them in.
  • Note that when elements are inserted to the dom, they do not require any special treatment but automatically gets initialized.
  • Note that the DOM Agent also runs once on document.ready event, so it also initialize element that appear in the dom to begin with.
