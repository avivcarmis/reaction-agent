The DOM Agent

The DOM Agent module satisfies two basic needs of a JS developer. The first main feature supports and generates jQuery based DOM change event. Meaning, if you only use jQuery methods to make DOM changes (e.g. html, append, remove, etc…) the DOM Agent will trigger DOM change events for any element who's inner DOM has been changed.

The second and more interesting feature of the DOM Agent is the DOM element constructor capability. Meaning, you can add your custom CSS selectors and a construction function to handle and create any Javascript functionality for any element inserted at any time to the document and matches the selector.

For example, suppose we would like to have an element that changes its color every time a document.click event is triggered. We would probably write a function to run on document.ready event that looks about that:

$(document).click(function() { var randomColor = someRandomColorGenerator(); // some function to return random css color value $('.colorful'). // select the 'colorful' element of ours css('background', randomColor); // set its color });

This example would work as expected. Now, suppose we have an AJAX application and we would like to dynamically insert another element just like that to the page. Our main options to go about it are either to add a script tag in the DOM below each one of these element tags, which is a horrible practice, since it generates massive code duplications and would be very hard to maintain and locate in the code. Another option that pops to mind is to have an initialize function, let's call it colorfulInit(), and just call it after every AJAX call. This would be a better practice, but if we consider the fact that we usually have web apps that contain many element that each demands its own initialization, like carousels, galleries, zoom elements, tooltips and right about every JS plugin that we would like to have in our code. In that situation, we need to have a bunch of different init functions, and we need to run each after every AJAX call. This, though, would be a bad practice, since we should keep in mind which new element types get created every AJAX call, if we add a new element type to a certain AJAX response, we should remember to add its initializer function to the AJAX call – when the word remember comes to mind, we should get ready to debug our code endlessly.

DOM Element Constructor

The DOM Agent suggest a simple and super convenient solution to that – with it, we can define DOM elements constructor once in our code, like that:

DOMAgent.add('.colorful', function() { var randomColor = someRandomColorGenerator(); // some function to return random css color value $(this). // select the 'colorful' element of ours css('background', randomColor); // set its color });

Note that without the agent you should keep track of which elements got initialized and skip them, the agent will call the constructor function only for elements that wasn't initialized, and will do that in the most efficient way by keeping track of DOM changes and locating only the DOM created, rather than scanning the entire DOM every change.

The API

Add
DOMAgent.add(selector, constructor);
What?

Defines a new constructor function by the given CSS selector.

Returns?

Returns a constructor usage key to be used in case you want to stop the constructor activity by the DOMAgent.remove function later.

Parameter Name Expected Type Default Value Description
selector String A string CSS selector to match elements in the DOM
constructor Function The constructor function to be applied for each of the matching elements in the DOM.
Note that the function will not receive any parameters, and the function scope will be the constructed DOM element, i.e. the this variable.
Example?
DOMAgent.add('.colorful', function() { var randomColor = someRandomColorGenerator(); // some function to return random css color value $(this). // select the 'colorful' element of ours css('background', randomColor); // set its color });
Remove
DOMAgent.remove(usageKey);
What?

Removes the constructor from the Agent. Note that all elements that already have been constructed will not be altered by this, but, new elements that matches the selector and get inserted to the DOM will not be constructed.

Returns?

True upon success or false if a constructor with the given usage key was not found.

Parameter Name Expected Type Default Value Description
usageKey Integer A constructor usage key that was received by the DOMAgent.add call
Example?
var usageKey = DOMAgent.add('.colorful', function() { var randomColor = someRandomColorGenerator(); // some function to return random css color value $(this). // select the 'colorful' element of ours css('background', randomColor); // set its color }); DOMAgent.remove(usageKey);

Events

The DOM Agent demands some resources in order to perform; Those resources ought to be minimized if the Agent is not being used, so, the events mechanism will be active in either or two cases. The first of which is upon a call to DOMAgent.add function, in that case the events mechanism becomes active and available for use. The other case comes in handy if you do not want to use the DOMAgent directly, but you just want to get notified on a change in an inner DOM of an element in your HTML structure, in that case, you must first place a call to DOMAgent.init() method to enable the events mechanism, and then use it anyway you like!

Example?
$("some-element").on('domagentchange', function() { // will be triggered each time the source element's inner HTML gets changed by any jQuery method });
