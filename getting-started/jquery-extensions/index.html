jQuery Extensions

The Reaction Agent framework comes with a few built-in utilities based on jQuery specified below.

Has Ancestor
$("some-element").hasAncestor(selector);
What?

Receive a string element selector and checks for existence of a matching ancestor.

Returns?

True if the element has an ancestor with a matching selector, false otherwise.

Parameter Name Expected Type Default Value Description
selector String A string CSS selector to match
Example?
<div id="parent"> <div id="some-id"> <div id="child"></div> </div> </div>$("#child").hasAncestor("#parent"); // returns true $("#child").hasAncestor("#some-id"); // returns true $("#child").hasAncestor("#child "); // returns true $("#child").hasAncestor("#some-other-thing"); // returns false
Dimension
$("some-element").getWidth([padding = false, [border = false, [margin = false]]]); $("some-element").getHeight([padding = false, [border = false, [margin = false]]]);
What?

The getWidth and getHeight methods are designed to bypass jQuery native dimension testing functions for two reasons. The first reason is simplicity, instead of browsing the jQuery documentation for 3 (or more) methods for each dimension (width, innerWidth, outerWidth) and choosing the correct one, the getWidth and getHeight methods concentrate all that functionality with a simple API determining whether or not to include padding, border or margin.
The second and more important reason, is the fact that jQuery native dimension calculators fail in case the element is not visible. Meaning, if, for example, the elements parent is hidden, then the tests returns 0. The getWidth and getHeight methods bypass that and calculates the element dimension as if it was visible.

Returns?

The width or height of the element respectively.

Parameter Name Expected Type Default Value Description
padding Boolean false Whether or not to include the element padding
border Boolean false Whether or not to include the element border
margin Boolean false Whether or not to include the element margin
Show And Test
$("some-element").showAndTest(callback);
What?

Designed to perform a test on an element that might be hidden and must be visible for the test. This method is used to calculate elements width and height for the getWidth and getHeight methods.

Returns?

The jQuery object.

Parameter Name Expected Type Default Value Description
callback Function A callback function to run only when the element is visible
Example?
$("some-element").showAndTest(function() { alert($(this).offset().top); // will alert the element offset from the top of the document, but only after validating that the element is visible. Afterwards the element will revert into its original state. });
Get CSS Restorer
$("some-element").getCSSRestorer(cssProperty1, cssProperty2, cssProperty3, ...);
What?

Used to restore some CSS properties of the element the current state without changing or overriding non-inline rules.
Receive a variadic number of arguments, each of them, a string representing a CSS property and returns the element current CSS restoring object.

Returns?

An object representing the current state of the element for the given CSS properties.

Example?
<style> .some-class { margin: 13px; } </style> <div id="element" class="some-class" style="background: #ccc;"></div>// element current style attribute value = "background: #ccc;" var restorer = $("#element").getCSSRestorer("margin", "padding"); $("#element").css("padding", "15px"); $("#element").css("background", "#000"); $("#element").css("margin", "17px"); // element current style attribute value = "background: #000; padding: 15px; margin: 17px;" $("#element").css(restorer); // element current style attribute value = "background: #000;"
