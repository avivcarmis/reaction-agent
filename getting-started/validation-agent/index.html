The Validation Agent

The Validation Agent is a mechanism to easily perform UX-oriented client side validation to form elements and mainly input elements. The agent takes a function or a regular expression to test the value and handler function to be triggered when the test successes of fails.

The agent is triggered directly on a set of elements via jQuery.

The API

validationAgent
$("some-element").validationAgent(tester, success, fail, [settings]);
What?

Receive validation parameters and sets them to validate the element value.

Returns?

The jQuery object.

Parameter Name Expected Type Default Value Description
tester Function / Object (regular expression) A test to validate the value of the element. Can be sent a function to receive a value and return true if the value is legal or false otherwise, or a Javascript regular expression
success Function A function to be triggered when the element value is legal
fail Function A function to be triggered when the element value is illegal
settings Object 0 Specified below
Available Settings
Parameter Name Expected Type Default Value Description
validateBlur Boolean true Whether or not to perform validation on element blur event
validateType Boolean true Whether or not to perform validation on element keypress event and block typing if an illegal value will arise
Example?
$("some-element").validationAgent(function(value) { return value != ''; // validate not empty }, function() { $(this).popover('hide'); // hide the popover message to indicate the value is legal }, function() { $(this).popover('show'); // show the popover message to indicate the value is illegal });
Pause
$("some-element").validationAgent("pause");
What?

Pauses the activity of the agent.

Returns?

The jQuery object.

Resume
$("some-element").validationAgent("resume");
What?

Resumes the activity of the agent.

Returns?

The jQuery object.

Usage Tip

If you are going to use the Validation Agent, you may want to consider integrating it into the DOM Agent so that you won't have to initialize each element in Javascript manually. Here's an example of how to do so:

// registering validation elements to the DOM Agent DOMAgent.add('[data-validation]', function() { var regexp = new RegExp($(this).attr('data-validation')); $(this).validationAgent(regexp, function() { $(this).popover('show'); }, function() { $(this).popover('destroy'); }); });

What we did in this piece of code is telling the DOM Agent that each element inserted to the DOM containing a data-validation attribute should be initialized to the validation agent. We do that using the regular expression value that is specified in the data-validation attribute, in case the regexp match test is failed, the element bootstrap popover is shown, or hidden otherwise.

Now, we can add element to the DOM like so:

<input type="text" name="username" value="" placeholder="Your username here" data-validation="^[a-zA-Z0-9]+$" title="Username" data-content="Username may contain only alpha numeric characters"> <input type="password" name=" password" value="" placeholder="Your password here" data-validation="^(?=.*\d)(?=.*[a-z])(?=.*[A-Z])[0-9a-zA-Z]{8,}$" title="Password" data-content="Password may contain at least one digit, one lower case letter, one upper case letter and at least 8 total characters">

Note that these elements will be automatically initialized as validation elements, whenever an illegal value will be typed into them, the typing will be blocked, and the message appear in the data-content will be popped over the element.

Important Note This piece of code is dependent on bootstrap framework.
