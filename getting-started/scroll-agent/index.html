The Scroll Agent

The Scroll Agent module was designed to easily control the scrolling events of your page. The agent is based on two basic mechanisms, the first of which are the scrollup and scrolldown events for detecting page scroll upwards and downwards respectively.

The second mechanism uses a concept of scrolling breakpoint, in which every breakpoint holds a ScrollTop value and a handler function to be triggered when window scroll value reached the ScrollTop value of the breakpoint.

Let's Make It Simple!

This page is a good example of the operation of the ScrollAgent! If your'e viewing this page from your desktop, laptop or tablet, look at the affix menu on the right side - now scroll up and down and see that the purple indicator changes it's state - moving from one item to another, depending on the current location of the window scroll value. but! wait a minute. If you scroll the page all the way up, and then scroll it a bit down, you'll see another thing happen, the header of the page gets minimized. Note that these are two different tasks, each creates it's own instance of the ScrollAgent, and then supply it with information about how to react and to what breakpoints in the page.

First of all, in order to create an instance of the ScrollAgent we must call it's constructor like this for example: var scrollAgentInstance = new ScrollAgent(); Then, the scrollAgentInstance variable is an instance of the ScrollAgent and we can use the ScrollAgent API as detailed below.

The API

Add
scrollAgentInstance.add(test, handler, [offset = 0]);
What?

Adds a new breakpoint to the agent.

Returns?

A breakpoint usage key to be used to remove the breakpoint later.

Parameter Name Expected Type Default Value Description
test Number / String / Function / Object A test to determine the ScrollTop value of the breakpoint. Can either be a number to set a fixed ScrollTop value, a function to dynamically return a ScrollTop value, or a string / object to identify a certain element and responsively determine the ScrollTop value to be the element top position. In the last case a string is used as a selector and object will either be a DOM element or a jQuery object
handler Function The handler function to be triggered when the breakpoint is activated
offset Number 0 When defining the test with an element, the offset parameter can be used to add or subtract a certain offset from the breakpoint ScrollTop value. E.g. when the breakpoing starts at the element below the header, but we want to to start 20px after
Example?
// Creating a ScrollAgent instance var sa = new ScrollAgent(); // adding a breakpoint to the top of the screen sa.add(0, function() { $('some-element').someAction(); }); // adding a breakpoint with a dynamic function sa.add(function() { return $(window).height() // responsively set the breakpoint value to bethe height of the window }, function() { $('some-element').someAction(); }); // adding a breakpoint with a string selector and a function name function doSomethingCool() { $('some-element').someAction(); } sa.add("footer", doSomethingCool); // will trigger doSomethingCool function when the footer top reaches the top of the window // adding a breakpoint with a jQuery object, a function name and an offset sa.add($("footer"), doSomethingCool, 100); // will trigger doSomethingCool function when the top of the window reaches 100 pixels below the footer top
Remove
scrollAgentInstance.remove(usageKey);
What?

Removes a breakpoint from the agent.

Returns?

True upon success or false in case the breakpoint wasn't found.

Parameter Name Expected Type Default Value Description
usageKey Number The usage key of the breakpoint to be removed

Events

To activate the events mechanism, the function ScrollAgent.events() should be called. The reason the mechanism is not activated by default is to minimize the agent footprint and prevent unnecessary tracking of the window scroll.
The Scrolling events are triggered on the document and the window, and contain the scrollup and scrolldown events.

Note the fact that the events is a static method of the ScrollAgent, meaning it is not to be used on an instance of the ScrollAgent like the rest of the API, but exactly like in the example below.

Example?
ScrollAgent.events(); $(window).on('scrollup', function() { console.log('window scrolled up'); }); $(document).on('scrolldown', MethodFactory.log('window scrolled down')); // MethodFactory.log is a utility method of reaction agent to return a function that logs a message to console.
